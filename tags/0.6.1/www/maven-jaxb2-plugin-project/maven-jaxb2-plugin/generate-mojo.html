jaxb2:generate

Full name:

org.jvnet.jaxb2.maven2:maven-jaxb2-plugin:0.6.1:generate

Description:

JAXB 2.x Mojo. @author Aleksei Valikov (valikov@gmx.net)

Attributes:

  • Requires a Maven 2.0 project to be executed.
  • Requires dependency resolution of artifacts in scope: runtime.
  • Binds by default to the lifecycle phase: generate-sources.

Required Parameters

Name Type Since Description
localRepository ArtifactRepository - (no description)
schemaDirectory File - The source directory containing *.xsd schema files. Notice that binding files are searched by default in this deriectory. Default value is: src/main/resources.
generateDirectory File -

Generated code will be written under this directory.

For instance, if you specify generateDirectory="doe/ray" and generatePackage="org.here", then files are generated to doe/ray/org/here.

Default value is: ${project.build.directory}/generated-sources/xjc.

Optional Parameters

Name Type Since Description
artifactResolver ArtifactResolver - (no description)
artifactFactory ArtifactFactory - Used internally to resolve plugins to their jar files.
mavenProjectBuilder MavenProjectBuilder - Artifact factory, needed to download source jars.
schemaLanguage String - Type of input schema language. One of: DTD, XMLSCHEMA, RELAXNG, RELAXNG_COMPACT, WSDL, AUTODETECT. If unspecified, it is assumed AUTODETECT.
schemaIncludes String[] -

A list of regular expression file search patterns to specify the schemas to be processed. Searching is based from the root of schemaDirectory.

If left udefined, then all *.xsd files in schemaDirectory will be processed.
schemaExcludes String[] - A list of regular expression file search patterns to specify the schemas to be excluded from the schemaIncludes list. Searching is based from the root of schemaDirectory.
bindingDirectory File -

The source directory containing the *.xjb binding files.

If left undefined, then the schemaDirectory is assumed.
bindingIncludes String[] -

A list of regular expression file search patterns to specify the binding files to be processed. Searching is based from the root of bindingDirectory.

If left undefined, then all *.xjb files in schemaDirectory will be processed.
bindingExcludes String[] - A list of regular expression file search patterns to specify the binding files to be excluded from the bindingIncludes. Searching is based from the root of bindingDirectory.
disableDefaultExcludes boolean - If 'true', maven's default exludes are NOT added to all the excludes lists. Default value is: false.
catalog File - Specify the catalog file to resolve external entity references (xjc's -catalog option)

Support TR9401, XCatalog, and OASIS XML Catalog format. See the catalog-resolver sample and this article for details.
catalogResolver String - Provides the class name of the catalog resolver. @return Class name of the catalog resolver.
generatePackage String -

The generated classes will all be placed under this Java package (xjc's -p option), unless otherwise specified in the schemas.

If left unspecified, the package will be derived from the schemas only.
readOnly boolean - If 'true', the generated Java source files are set as read-only (xjc's -readOnly option). Default value is: false.
extension boolean - If 'true', the XJC binding compiler will run in the extension mode (xjc's -extension option). Otherwise, it will run in the strict conformance mode. Default value is: false.
strict boolean - If 'true', Perform strict validation of the input schema (xjc's -nv option). Default value is: true.
writeCode boolean - If 'false', the plugin will not write the generated code to disk. Default value is: true.
verbose boolean -

If 'true', the plugin and the XJC compiler are both set to verbose mode (xjc's -verbose option).

It is automatically set to 'true' when maven is run in debug mode (mvn's -X option).

Default value is: false.
debug boolean -

If 'true', the XJC compiler is set to debug mode (xjc's -debug option)and the 'com.sun.tools.xjc.Options.findServices' property is set, to print any add-on instanciation messages.

It is automatically set to 'true' when maven is run in debug mode (mvn's -X option).

Default value is: false.
args String> -

A list of extra XJC's command-line arguments (items must include the dash '-').

Arguments set here take precedence over other mojo parameters.
forceRegenerate boolean - If 'true', no up-to-date check is performed and the XJC always re-generates the sources. Default value is: false.
removeOldOutput boolean -

If 'true', the [generateDirectory] dir will be deleted before the XJC binding compiler recompiles the source files.

Note that if set to 'false', the up-to-date check might not work, since XJC does not regenerate all files (i.e. files for "any" elements under 'xjc/org/w3/_2001/xmlschema' directory).

Default value is: true.
otherDepends String[] - A list of of input files or URLs to consider during the up-to-date. By default it always considers: 1. schema files, 2. binding files, 3. catalog file, and 4. the pom.xml file of the project executing this plugin.
episodeFile File - (no description) Default value is: ${project.build.directory}/generated-sources/xjc/META-INF/sun-jaxb.episode.
episode boolean - (no description) Default value is: true.
plugins Artifact[] - XJC plugins to be made available to XJC. They still need to be activated by using <args> and enable plugin activation option.
episodes Artifact[] - (no description)
