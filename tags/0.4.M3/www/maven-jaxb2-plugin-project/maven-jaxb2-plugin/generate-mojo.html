jaxb2:generate

A mojo that uses JAXB 2.x XJC compiler to generate java source classes from schemas (XML schemas, DTD, WSDL, or RELAXNG). For details on JAXB see JAXB 2.x Project.

Parameters for the goal:

Parameter Type Expression Default Value Description
args (Optional) List - -

A list of extra XJC's command-line arguments (items must include the dash '-').

Arguments set here take precedence over other mojo parameters.

artifactFactory (Optional) (Discovered) ArtifactFactory - - No description.
artifactResolver (Optional) (Discovered) ArtifactResolver - - Used internally to resolve {@link #plugins} to their jar files.
bindingDirectory (Optional) File ${maven.xjc2.bindingDirectory} -

The source directory containing the *.xjb binding files.

If left undefined, then the schemaDirectory is assumed.

bindingExcludes (Optional) String - - A list of regular expression file search patterns to specify the binding files to be excluded from the bindingIncludes. Searching is based from the root of bindingDirectory.
bindingIncludes (Optional) String - -

A list of regular expression file search patterns to specify the binding files to be processed. Searching is based from the root of bindingDirectory.

If left undefined, then all *.xjb files in schemaDirectory will be processed.

catalog (Optional) File ${maven.xjc2.catalog} -

Specify the catalog file to resolve external entity references (xjc's -catalog option)

Support TR9401, XCatalog, and OASIS XML Catalog format. See the catalog-resolver sample and this article for details.

classpathElements List ${project.compileClasspathElements} - Project classpath. Used internally when runing the XJC compiler.
debug (Optional) boolean ${maven.xjc2.debug} false

If 'true', the XJC compiler is set to debug mode (xjc's -debug option)and the 'com.sun.tools.xjc.Options.findServices' property is set, to print any add-on instanciation messages.

It is automatically set to 'true' when maven is run in debug mode (mvn's -X option).

disableDefaultExcludes (Optional) boolean ${maven.xjc2.disableDefaultExcludes} false If 'true', maven's default exludes are NOT added to all the excludes lists.
extension (Optional) boolean ${maven.xjc2.extension} false If 'true', the XJC binding compiler will run in the extension mode (xjc's -extension option). Otherwise, it will run in the strict conformance mode.
forceRegenerate (Optional) boolean ${maven.xjc2.forceRegenerate} false If 'true', no up-to-date check is performed and the XJC always re-generates the sources.
generateDirectory File ${maven.xjc2.generateDirectory} ${project.build.directory}/generated-sources/xjc

Generated code will be written under this directory.

For instance, if you specify generateDirectory="doe/ray" and generatePackage="org.here", then files are generated to doe/ray/org/here.

generatePackage (Optional) String ${maven.xjc2.generatePackage} -

The generated classes will all be placed under this Java package (xjc's -p option), unless otherwise specified in the schemas.

If left unspecified, the package will be derived from the schemas only.

localRepository ArtifactRepository ${localRepository} - No description.
mavenProjectBuilder (Optional) (Discovered) MavenProjectBuilder - - Artifact factory, needed to download source jars.
otherDepends (Optional) String - - A list of of input files or URLs to consider during the up-to-date. By default it always considers: 1. schema files, 2. binding files, 3. catalog file, and 4. the pom.xml file of the project executing this plugin.
plugins (Optional) Artifact - - XJC plugins to be made available to XJC. They still need to be activated by using <args> and enable plugin activation option.
project MavenProject ${project} - The Maven project reference, used internally.
readOnly (Optional) boolean ${maven.xjc2.readOnly} false If 'true', the generated Java source files are set as read-only (xjc's -readOnly option).
removeOldOutput (Optional) boolean ${maven.xjc2.removeOldOutput} true

If 'true', the [generateDirectory] dir will be deleted before the XJC binding compiler recompiles the source files.

Note that if set to 'false', the up-to-date check might not work, since XJC does not regenerate all files (i.e. files for "any" elements under 'xjc/org/w3/_2001/xmlschema' directory).

schemaDirectory File ${maven.xjc2.schemaDirectory} src/main/resources The source directory containing *.xsd schema files. Notice that binding files are searched by default in this deriectory.
schemaIncludes (Optional) String - -

A list of regular expression file search patterns to specify the schemas to be processed. Searching is based from the root of schemaDirectory.

If left udefined, then all *.xsd files in schemaDirectory will be processed.

schemaLanguage (Optional) String ${maven.xjc2.schemaLanguage} -

Type of input schema language. One of: DTD, XMLSCHEMA, RELAXNG, RELAXNG_COMPACT, WSDL, AUTODETECT

If unspecified, it is assumed AUTODETECT.

schemasExcludes (Optional) String - - A list of regular expression file search patterns to specify the schemas to be excluded from the schemaIncludes list. Searching is based from the root of schemaDirectory.
strict (Optional) boolean ${maven.xjc2.strict} true If 'true', Perform strict validation of the input schema (xjc's -nv option).
verbose (Optional) boolean ${maven.xjc2.verbose} false

If 'true', the plugin and the XJC compiler are both set to verbose mode (xjc's -verbose option).

It is automatically set to 'true' when maven is run in debug mode (mvn's -X option).

writeCode (Optional) boolean ${maven.xjc2.writeCode} true If 'false', the plugin will not write the generated code to disk.
